# GrapeRoot Dual-Graph

## What It Is

Local MCP server that builds a dual graph of your codebase — an Info Graph (structure, imports, dependencies) and an Action Graph (session memory of reads/edits). Claims to reduce hallucinations and token waste by giving Claude Code a persistent map. https://grape-root.vercel.app/

## Problem It Solves

Claude Code starts cold every session. GrapeRoot provides persistent codebase understanding and session memory across sessions via MCP.

## Key Observations

- Concept is sound — codebase graph indexing via MCP is a real and growing category
- **No public GitHub repo** — source code is not available anywhere
- Solo developer (kunal12203, 1 follower, mostly small projects) — no track record with security-sensitive tooling
- Closed-source binary that gets full read access to your codebase and runs a local server
- Landing page is polished but the substance behind it is thin
- Website self-describes as "generated by Claude Code + GrapeRoot" — weekend project vibes

## Verdict

**Skip.** Would not install a closed-source binary with codebase access from an unverified developer. The concept is worth pursuing via open-source alternatives:
- [CodeGraphContext](https://github.com/CodeGraphContext/CodeGraphContext) — CLI + MCP server, open source
- [codebase-memory-mcp](https://github.com/DeusData/codebase-memory-mcp) — 64 languages, single Go binary, open source
- [mcp-server-codegraph](https://github.com/CartographAI/mcp-server-codegraph) — graph representation, open source
